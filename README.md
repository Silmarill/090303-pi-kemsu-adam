# Проект Адам. Этап 1 - Астероидная зона

Галактика живёт **хронами** — дискретными тактами времени. В аномальных зонах рождаются астероиды, содержащие загадочное вещество **Echos**. Каждый хрон астероиды теряют часть ресурса; если не успеть его извлечь — он исчезает навсегда. Ваша задача — создать консольную симуляцию, которая по нажатию клавиши отмеряет хроны, управляет астероидами через пул объектов и отображает их состояние.

В проекте реализуются два ключевых паттерна: **Наблюдатель** (интерфейс `IChroneListener` + статический `ChroneManager`) для синхронизации объектов с тактами, и **Пул объектов** (класс `AsteroidEmitter` с очередью `Queue<Asteroid>`) для переиспользования астероидов вместо их постоянного создания и удаления.

---

## 🎮 Основные элементы

- **Хроны** – нажатие `Enter` продвигает время на один такт, `Esc` – выход.
- **Астероиды** – имеют ресурс `Echos`, состояние `Idle` / `Depleted`, уникальные идентификаторы создания и спавна.
- **Пул** – хранит свободные астероиды в очереди; методы `Spawn()` и `Recycle()` управляют их жизненным циклом.
- **Активные астероиды** – хранятся в `List<Asteroid>`; каждый хрон их ресурс уменьшается на 100; при достижении 0 астероид возвращается в пул.
- **Спавн** – на 0-м хроне появляется 3 астероида, затем каждые 5 хронов – от 1 до 3 новых.

---

## 🕹️ Управление

| Клавиша | Действие               |
|---------|------------------------|
| `Enter` | Следующий хрон (такт)  |
| `Esc`   | Завершение программы   |

---

## 🌟 Усложнения (по вариантам)

1. **Два эмиттера с разными интервалами** – астероиды спавнятся каждые 3 и 7 хронов.
2. **Чёрная дыра** – с вероятностью 5% все активные астероиды теряют 10% ресурса.
3. **Ресурсный бум** – с вероятностью 5% вне графика появляется астероид с повышенным ресурсом.
4. **Цветной вывод** – окраска астероидов в зависимости от процента оставшегося ресурса.
5. **Сохранение в XML** – по клавише `L` сохраняется состояние, при старте – загрузка.
6. **Долгая утилизация** – астероиды проходят 2-тактовую очередь утилизации перед возвратом в пул.
7. **Логгер (Singleton)** – отладочные сообщения по флагу, включение/выключение по `D`.
8. **Типы астероидов** – разные типы теряют ресурс с разной скоростью.
9. **Аномалии** – временные эффекты, изменяющие поведение эмиттера или деградацию.
10. **Статистика и мониторинг** – по клавише `S` вывод статистики (максимум активных астероидов, общее количество созданных и т.д.).
